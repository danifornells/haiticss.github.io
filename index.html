Just another baseline perspective to wear any project with the ITCSS architecture. Mainly based on the excellent project inuitcss by Harry Roberts, from where we found our best suitable starting point.

The idea to create this project started from the necessity to add some modifications and missing features to inuitcss that probably would not fit into the original project roadmap. In the end we added some layers, and a visual language into the architecture, helping to build a design system out of the box.

Main differences from inuitcss are:

  • Number prefixes on CSS layers, helping non familiar developers to get into the architecture hierarchy.
  • Design foundations with a visual language.
  • Namespaced variables to improve compatibility when importing into an existing project.
  • Feature toggling variables to include/exclude any object or utility and prevent unused CSS output.
  • Objects defined as mixin as well to use the magic on components and change the layout throughout breakpoints.
  • Added an abstract animations layer.
  • Added a color schemes layer.
  • Added a material layer with base elements, not components, with configuration map variables.
  • Scoped version feature, helping to start old site migrations on the fly, respecting the legacy code, but breaking most risky inheritance.

The project is not mature yet to be used on any production project, but if you're curious, feel free to browse our repository and get an overview. Hope you like it 😜
